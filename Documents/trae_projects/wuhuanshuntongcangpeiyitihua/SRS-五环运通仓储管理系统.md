## 1. 引言

### 1.1 编写目的

本文档旨在详细描述五环运通仓储管理系统的功能需求、非功能需求及业务规则，为项目开发、测试和评审提供完整、准确的需求依据。本文档适用于系统开发团队、测试团队、项目管理人员及相关利益方。

### 1.2 项目背景

五环运通仓储管理系统是一个面向冷链物流行业的仓储管理平台，系统提供货物管理、仓库管理、货龄提醒等功能，支持冷冻、冷藏、常温等多种仓库类型的统一管理。系统采用现代化的前端技术栈（React + TypeScript + Vite），确保良好的用户体验和系统性能。

### 1.3 术语定义

| 术语 | 定义 |
|------|------|
| 货物 | 存储在仓库中的商品单位 |
| 仓库 | 用于存储货物的物理空间，分为冷冻、冷藏、常温三种类型 |
| 保质期 | 货物从生产日期开始的有效存储天数 |
| 货龄 | 货物从生产日期到当前日期的实际天数 |
| 货龄提醒 | 当货物货龄超过预设天数时向用户发出的风险提示 |
| CRUD | Create（创建）、Read（读取）、Update（更新）、Delete（删除）四种基本数据操作 |

## 2. 系统概述

### 2.1 系统目标

本系统旨在为五环运通提供一套完整的仓储管理解决方案，实现以下目标：

1. **货物管理**：建立完善的货物档案，支持货物信息的增删改查，关联保质期信息
2. **仓库管理**：管理多类型仓库，支持仓库定位信息维护
3. **货龄提醒**：建立货龄预警机制，在入库/出库业务中实现风险提醒
4. **用户体验**：提供简洁易用的操作界面，提升工作效率

### 2.2 用户角色

| 业务员 | 业务操作人员 | 货物查询、出入库单据操作 |

## 3. 功能需求

### 3.1 货物管理

#### 3.1.1 功能说明

货物管理模块用于维护系统中所有货物的基础信息档案，包括货物名称、编码、规格、单位、保质期等核心字段。货物档案是后续仓库管理和货龄提醒的基础数据来源。

#### 3.1.2 业务规则

1. 货物编码在整个系统中唯一，不可重复
2. 保质期以天为单位，支持0-9999天范围
3. 货物删除前需确认，无关联数据方可删除
4. 支持按货物名称、编码、规格关键字搜索

#### 3.1.3 页面描述

页面采用标准列表+弹窗表单布局：
- 顶部：搜索区域、操作按钮区
- 中部：数据表格展示区
- 底部：分页导航区
- 弹窗：新增/编辑表单

#### 3.1.4 查询条件

| 字段 | 类型 | 说明 |
|------|------|------|
| 关键字搜索 | 文本输入 | 搜索货物名称、编码、规格 |

#### 3.1.5 列表展示字段

| 字段 | 数据类型 | 可见性 | 默认显示 |
|------|----------|--------|----------|
| 序号 | 数字 | 固定 | 是 |
| 货物名称 | 字符串 | 可配置 | 是 |
| 货物编码 | 字符串 | 可配置 | 是 |
| 规格 | 字符串 | 可配置 | 是 |
| 单位 | 字符串 | 可配置 | 是 |
| 保质期（天） | 数字 | 可配置 | 是 |

#### 3.1.6 新增功能

- **触发方式**：点击「新增货物」按钮
- **表单字段**：
  - 货物名称（必填）
  - 货物编码（必填，唯一性校验）
  - 规格（选填）
  - 单位（必填，默认值：件）
  - 保质期（天）（选填，默认值：0）
- **交互流程**：填写表单 → 客户端校验 → 提交保存 → 关闭弹窗 → 刷新列表
- **重复处理**：货物编码重复时提示「货物编码已存在」

#### 3.1.7 编辑功能

- **触发方式**：点击列表「编辑」操作按钮
- **表单预填**：自动加载当前货物数据
- **交互流程**：编辑表单 → 校验 → 保存 → 刷新列表
- **编码变更**：编辑时可修改编码，但需校验唯一性

#### 3.1.8 删除功能

- **触发方式**：点击列表「删除」操作按钮
- **确认提示**：弹出确认对话框「确定要删除该货物吗？」
- **删除后**：刷新当前页列表，若当前页无数据则返回上一页

#### 3.1.9 查看功能

- **触发方式**：点击列表「查看」操作按钮
- **展示方式**：alert弹窗展示完整货物信息
- **展示内容**：货物名称、编码、规格、单位、保质期

#### 3.1.10 数据校验规则

| 字段 | 校验规则 | 错误提示 |
|------|----------|----------|
| 货物名称 | 非空，最大50字符 | 请填写货物名称 |
| 货物编码 | 非空，最大20字符，唯一性 | 请填写货物编码 / 货物编码已存在 |
| 规格 | 最大100字符 | - |
| 单位 | 非空，从预定义列表选择 | - |
| 保质期 | 0-9999整数 | - |

#### 3.1.11 页面交互逻辑

1. **搜索**：实时过滤，输入即搜索，无需点击查询按钮
2. **分页**：每页10条记录，支持页码跳转
3. **列设置**：支持用户自定义表格列的显示/隐藏
4. **导入导出**：支持Excel格式的批量导入导出功能
5. **排序**：按更新时间倒序排列

#### 3.1.12 异常处理

| 异常场景 | 处理方式 |
|----------|----------|
| 网络请求失败 | 显示友好错误提示 |
| 数据加载中 | 显示loading状态 |
| 无数据 | 显示「暂无货物数据」提示 |
| 搜索无结果 | 显示「未找到匹配的货物」 |

#### 3.1.13 权限控制

| 操作 | 权限要求 |
|------|----------|
| 查看列表 | 所有登录用户 |
| 新增货物 | 管理员、仓管员 |
| 编辑货物 | 管理员、仓管员 |
| 删除货物 | 仅管理员 |
| 导入导出 | 管理员 |

### 3.2 仓库管理

#### 3.2.1 功能说明

仓库管理模块用于维护系统中所有仓库的基础信息，支持仓库的增删改查操作。仓库类型分为冷冻、冷藏、常温三种，并支持定位信息维护（地址和经纬度）。

#### 3.2.2 业务规则

1. 仓库编码在系统中唯一，不可重复
2. 仓库类型必须从预定义列表中选择：冷冻、冷藏、常温
3. 定位信息为必填项，支持地址搜索和地图选点
4. 经纬度信息通过地图选点自动计算生成
5. 删除仓库前需二次确认

#### 3.2.3 页面描述

页面布局与货物管理类似，采用查询条件区、列表区、分页区的标准布局。

#### 3.2.4 查询条件

| 字段 | 类型 | 说明 |
|------|------|------|
| 仓库编码 | 文本输入 | 模糊匹配 |
| 仓库名称 | 文本输入 | 模糊匹配 |
| 仓库类型 | 下拉选择 | 全部类型/冷冻/冷藏/常温 |

#### 3.2.5 列表展示字段

| 字段 | 数据类型 | 说明 |
|------|----------|------|
| 序号 | 数字 | 自动生成 |
| 仓库编码 | 字符串 | 系统唯一 |
| 仓库名称 | 字符串 | - |
| 仓库类型 | 枚举 | 冷冻/冷藏/常温，显示为标签样式 |
| 定位信息 | 字符串 | 地址描述 |
| 操作 | 操作组 | 查看、编辑、删除 |

#### 3.2.6 新增功能

- **触发方式**：点击「新增仓库」按钮
- **表单字段**：
  - 仓库编码（必填，唯一性校验）
  - 仓库名称（必填）
  - 仓库类型（必填，下拉选择）
  - 定位信息（必填，支持地址搜索和地图选点）
  - 经纬度（自动生成，只读）
- **交互流程**：填写表单 → 校验 → 保存 → 刷新列表
- **编码重复**：提示「仓库编码已存在」

#### 3.2.7 编辑功能

- **触发方式**：点击列表「编辑」操作按钮
- **表单预填**：自动加载当前仓库数据
- **编辑限制**：仓库编码在编辑时不可修改
- **交互流程**：编辑表单 → 保存 → 刷新列表

#### 3.2.8 删除功能

- **触发方式**：点击列表「删除」操作按钮
- **确认提示**：弹出确认对话框 `确定要删除仓库 "${仓库名称}" 吗？`
- **删除后**：刷新列表，若当前页无数据则返回上一页

#### 3.2.9 查看功能

- **触发方式**：点击列表「查看」操作按钮
- **展示方式**：alert弹窗展示完整仓库信息
- **展示内容**：
  - 仓库编码、名称、类型
  - 位置（地址描述）
  - 纬度、经度
  - 创建时间、更新时间

#### 3.2.10 数据校验规则

| 字段 | 校验规则 | 错误提示 |
|------|----------|----------|
| 仓库编码 | 非空，最大20字符，唯一性 | 请输入仓库编码 / 仓库编码已存在 |
| 仓库名称 | 非空，最大50字符 | 请输入仓库名称 |
| 仓库类型 | 非空，从列表选择 | - |
| 定位信息 | 非空 | 请输入定位信息 |

#### 3.2.11 页面交互逻辑

1. **查询条件**：实时过滤，无需点击查询按钮
2. **重置按钮**：清空所有查询条件，恢复默认状态
3. **仓库类型标签**：不同类型显示不同颜色标签
   - 冷冻：蓝色标签

   - 点击「打开地图选择」按钮展开地图
   - 点击「随机选择位置」模拟地图选点

3. **经纬度**：
   - 由系统自动计算生成
   - 展示精度：小数点后6位
   - 只读显示，不可手动编辑

#### 3.2.13 异常处理

| 异常场景 | 处理方式 |

### 3.3 货物货龄提醒

#### 3.3.1 功能说明

货物货龄提醒模块用于配置货物的货龄提醒规则。当用户在入库单或出库单中保存单据时，系统根据配置的规则自动校验货物货龄，对超过提醒天数的货物进行风险提示。该功能仅提供提醒，不阻断业务操作。

#### 3.3.2 业务规则

1. 仅允许对已维护保质期（天）的货物配置货龄提醒规则
2. 同一SKU仅允许配置一条货龄提醒规则
3. 货龄计算公式：`货龄（天）= 单据创建日期 - 生产日期`
4. 触发条件：`货龄（天） > 货龄提醒天数`
5. 该功能仅进行风险提醒，不限制业务操作

#### 3.3.3 页面描述

页面采用标准规则配置列表布局，包括查询区、规则列表区、分页区。

#### 3.3.4 查询条件

| 字段 | 类型 | 说明 |
|------|------|------|
| 关键字搜索 | 文本输入 | 搜索货物编码或名称 |

#### 3.3.5 列表展示字段

| 字段 | 数据类型 | 说明 |
|------|----------|------|
| 货物编码 | 字符串 | 关联货物档案 |
| 货物名称 | 字符串 | 自动带出 |
| 保质期（天） | 数字 | 自动带出货物档案中的保质期 |
| 货龄提醒天数 | 数字 | 触发提醒的天数阈值 |
| 备注 | 字符串 | 选填 |
| 操作 | 操作组 | 编辑、删除 |

#### 3.3.6 新增功能

- **触发方式**：点击「新增规则」按钮
- **表单字段**：
  - 货物编码（必填，下拉选择，仅显示已维护保质期的货物）
  - 货物名称（自动带出，只读）
  - 保质期（天）（自动带出，只读）
  - 货龄提醒天数（必填，正整数）
  - 备注（选填）
- **交互流程**：选择货物 → 自动带出名称和保质期 → 设置提醒天数 → 保存
- **唯一性校验**：同一货物仅允许配置一条规则

#### 3.3.7 编辑功能

- **触发方式**：点击列表「编辑」操作按钮
- **编辑限制**：
  - 货物编码不可修改
  - 只能修改货龄提醒天数和备注
- **交互流程**：修改提醒天数/备注 → 保存 → 刷新列表

#### 3.3.8 删除功能

- **触发方式**：点击列表「删除」操作按钮
- **确认提示**：弹出确认对话框 `确定要删除货物 "${货物名称}" 的货龄提醒规则吗？`
- **删除后**：该SKU不再参与货龄校验

  alerts: StockAgeAlertDetail[]; // 超限货物明细列表

// 保存单据流程
async function handleSaveBill(billData: BillData) {
  // 1. 执行货龄校验
  const checkResult = checkStockAgeAlerts(billData.items, billData.documentDate);
  
  // 2. 如果存在超限货物
  if (checkResult.hasAlert) {
    // 3. 显示提醒弹窗
    showAlertModal(checkResult);
    
    // 4. 等待用户选择
    const userChoice = await waitForUserChoice();

    if (userChoice === 'cancel') {
      return;
    }
  }
  
  // 6. 执行保存操作
  await saveBill(billData);
}

### 5.3 数据关系说明

1. **货物与货龄提醒规则**：一对多关系
   - 一个货物可以配置多条货龄提醒规则（系统限制为一条）
   - 删除货物时需先删除关联的提醒规则

2. **仓库独立性**：仓库与货物为独立实体，无直接关联

3. **业务关联**：
   - 入库单/出库单包含货物明细
   - 货物明细关联货物档案获取保质期
   - 货物明细关联货龄提醒规则进行校验

## 7. 业务规则汇总

### 7.1 货物管理规则

| 规则编号 | 规则描述 | 优先级 |
|----------|----------|--------|
| GR-001 | 货物编码在整个系统中唯一，不可重复 | 高 |
| GR-002 | 保质期以天为单位，支持0-9999范围 | 中 |
| GR-003 | 删除货物前需确认，且无关联数据方可删除 | 高 |
| GR-004 | 保质期>0的货物可配置货龄提醒规则 | 高 |
| GR-005 | 货物列表按更新时间倒序排列 | 低 |

### 7.2 仓库管理规则

| 规则编号 | 规则描述 | 优先级 |
|----------|----------|--------|
| SR-001 | 仓库编码在整个系统中唯一，不可重复 | 高 |
| SR-002 | 仓库类型必须为：冷冻、冷藏、常温之一 | 高 |
| SR-003 | 定位信息为必填项 | 高 |
| SR-004 | 经纬度由系统自动计算生成，不可手动编辑 | 中 |
| SR-005 | 删除仓库前需二次确认 | 高 |
| SR-006 | 编辑仓库时编码不可修改 | 中 |

### 7.3 货龄提醒规则

| 规则编号 | 规则描述 | 优先级 |
|----------|----------|--------|
| AR-001 | 仅允许对已维护保质期的货物配置提醒规则 | 高 |
| AR-002 | 同一SKU仅允许配置一条货龄提醒规则 | 高 |
| AR-003 | 货龄计算：单据创建日期 - 生产日期 | 高 |
| AR-004 | 触发条件：货龄 > 货龄提醒天数 | 高 |
| AR-005 | 该功能仅提供风险提醒，不阻断业务操作 | 高 |
| AR-006 | 删除规则后该SKU不再参与校验 | 高 |
| AR-007 | 多超限货物统一弹窗提醒 | 中 |
| AR-008 | 每次保存均重新执行校验 | 高 |
